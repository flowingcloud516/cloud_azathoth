# 存储知识地图

## 核心要点

从上到下梳理存储领域涉及的知识体系，作为后续深入学习的索引。覆盖从应用层到底层介质的完整 I/O 路径，以及分布式、网络、性能工程等横向领域。

## 详细内容

### 一、应用层

- **关系型数据库**：MySQL（InnoDB 引擎、B+ 树、redo/undo log、binlog）、PostgreSQL（MVCC、WAL）、TiDB（Raft + 行存）
- **NoSQL / KV 存储**：RocksDB（LSM-Tree、Compaction、Bloom Filter）、LevelDB、MongoDB（WiredTiger）
- **缓存**：Redis（数据结构、持久化 RDB/AOF、内存管理）、Memcached（slab allocator）
- **对象存储**：S3 协议语义、MinIO（纠删码模式）、Ceph RGW
- **消息队列**：Kafka（segment 文件、零拷贝 sendfile、页缓存利用、ISR 机制）
- **大数据存储**：HDFS（NameNode/DataNode、块放置策略）、Parquet / ORC（列存格式、谓词下推）
- **用户态存储系统**：SPDK（NVMe 用户态驱动、轮询模式、无锁、vhost、iSCSI target、Blobstore）

### 二、系统层（I/O 接口与运行时）

- **I/O 模型对比**：Buffered I/O（利用页缓存）vs Direct I/O（O_DIRECT，绕过缓存）vs mmap（内存映射）
- **异步 I/O 接口**：
    - libaio —— Linux 原生 AIO，仅支持 O_DIRECT，限制多，生态老化
    - io_uring —— 共享环形队列，支持所有 I/O 类型，系统调用开销低，是现代方案
    - SPDK 轮询模式 —— 完全绕开内核，用户态完成 NVMe 命令提交
- **Page Cache**：radix tree 索引、预读（Readahead）、脏页回刷（writeback、pdflush 到内核线程）、水位控制（dirty_ratio / dirty_background_ratio）
- **内核 hints**：fadvise（WILLNEED / DONTNEED / SEQUENTIAL / RANDOM）、madvise、fsync / fdatasync 的取舍

### 三、内核层

- **VFS 层**：superblock、inode、dentry、file 对象的关系；lookup 流程；dcache / icache
- **文件系统**：
    - 通用：ext4（extent、journal 模式）、xfs（分配组、延迟分配、COW 重写）
    - 闪存优化：f2fs（GC、node/segment/direct/node 结构）、btrfs（COW、快照、校验和）
    - 网络文件系统：NFS v3/v4、CIFS/SMB
- **Block 层**：
    - bio 结构（向量 I/O、merge）、request_queue、plug/unplug 机制
    - I/O 调度器：mq-deadline（闪存简洁）、kyber（延迟目标）、bfq（带宽公平）
    - 多队列 blk-mq 架构
    - [[盘侧超时机制]]：I/O 超时路径、abort/reset/failover 的级联处理
- **Device Mapper**：dm-linear、dm-crypt（加密）、dm-thin（精简配置）、dm-cache、dm-multipath、LVM
- **MD**：软 RAID（RAID 0/1/5/6/10）、write hole 问题
- **FUSE**：用户态文件系统框架（s3fs、rclone mount）
- **非易失内存**：mmap DAX 直连、/dev/pmem、fsdax vs devdax

### 四、传输协议与网络层

- **块存储协议**：
    - NVMe：PCIe 直连，SQ/CQ 队列对，PRP/SGL
    - NVMe-oF：基于 Fabrics 扩展，支持 RDMA/TCP/FC 传输
    - SCSI / SAS / FC：传统企业存储协议栈
- **文件存储协议**：NFS、CIFS/SMB
- **RDMA**：InfiniBand、RoCE v1/v2、iWARP —— [[NVMe RDMA 框架]]
- **内核网络栈**：TCP/IP 在存储路径中的开销，用户态网络绕过
- **多路径**：dm-multipath、ALUA、原生 NVMe multipath

### 五、介质层

- **HDD 基础**：盘片/磁道/扇区、寻道时间、旋转延迟、SMR（瓦记录）
- **SSD 内部**：
    - NAND 类型：SLC / MLC / TLC / QLC / PLC（寿命、速度、价格梯度）
    - 3D NAND 堆叠
    - FTL（Flash Translation Layer）：地址映射、磨损均衡、垃圾回收（GC）、写放大（WAF）
    - 数据完整性：DIF（T10 PI）、LDPC ECC、端到端数据保护
    - 接口：SATA、SAS、NVMe（PCIe）；外形：M.2、U.2 / U.3、EDSFF
- **新兴介质与架构**：
    - ZNS SSD（Zoned Namespaces）：主机管理数据放置，降低写放大，降低 OP
    - FDP（Flexible Data Placement）：NVMe 2.0 标准，灵活数据放置
    - Open Channel SSD：主机管理 FTL
    - SCM / 持久内存：Intel Optane、CXL.mem
    - 冷存储：磁带（LTO）、光盘、高密度 HDD
- **可观测性**：SMART 指标、NVMe 遥测、介质寿命评估

### 六、数据服务层

- **数据保护**：RAID 级别与取舍、纠删码（Reed-Solomon）vs 多副本、快照（COW/ROW）、克隆、远程复制
- **数据缩减**：在线/离线压缩（LZ4/ZSTD）、在线/后处理去重、零块检测、delta 复制
- **空间管理**：精简配置、UNMAP/TRIM/DISCARD
- **QoS**：I/O 限流（cgroup blkio）、令牌桶/漏桶、优先级队列

### 七、分布式存储

- **一致性协议**：Paxos、Raft（多数派写入、日志复制、snapshot）、Gossip
- **架构模式**：Shared-Nothing vs Shared-Disk、控制面/数据面分离
- **数据放置**：一致性哈希（一致性哈希环、虚拟节点）、CRUSH 算法
- **复制与恢复**：主从复制、链式复制、Quorum 读写、故障域隔离、rebalance
- **具体系统**：Ceph（RADOS/Crush/OSD/Monitor）、GlusterFS、MinIO

### 八、虚拟化与容器

- **虚拟化存储**：qcow2 / vmdk 格式、virtio-blk / virtio-scsi、vhost-user（SPDK 加速）
- **容器存储**：CSI（Container Storage Interface）规范、PV/PVC 抽象、存储类（StorageClass）
- **云存储抽象**：弹性块存储（EBS 类）、对象存储（S3 类）、文件存储（EFS/NFS 类）

### 九、性能工程

- **基准测试工具**：fio（参数组合：bs/iodepth/numjobs/ioengine/direct/rate）、Vdbench、Flexible I/O
- **I/O 模式分析**：随机/顺序、读写比例、延迟敏感 vs 吞吐优先、突发 vs 稳态
- **关键指标**：IOPS、带宽、P50/P99/P999 延迟、写放大因子（WAF）、CPU 效率（IOPS per core）
- **可观测性体系**：iostat、blktrace/btt、bpftrace（bio 延迟分布）、perf flamegraph、eBPF 热力图
